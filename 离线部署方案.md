# SQL Lint Service 离线环境Linux部署方案（用户目录版）

## 一、环境准备

### 1.1 系统要求
- **操作系统**: Linux (CentOS 7+/Ubuntu 18.04+)
- **Python版本**: 3.12.x
- **部署路径**: `/home/$USER/sql-lint-service/`
- **内存**: 至少2GB
- **磁盘空间**: 至少500MB

## 二、离线依赖包准备（在可联网环境执行）

### 2.1 导出依赖文件
```bash
# 进入项目目录
cd sql-lint-service

# 使用poetry导出requirements.txt
poetry export -f requirements.txt --output requirements.txt --without-hashes

# 查看导出的依赖
cat requirements.txt
```

### 2.2 下载离线依赖包
```bash
# 创建离线包目录
mkdir -p ./offline-packages

# 下载所有依赖包（针对Linux x86_64平台）
pip download -r requirements.txt \
  -d ./offline-packages \
  --platform manylinux2014_x86_64 \
  --python-version 312 \
  --only-binary=:all: \
  --no-deps

# 检查下载的包
ls -lh ./offline-packages/ | wc -l
du -sh ./offline-packages/
```

### 2.3 打包传输文件
```bash
# 创建部署包
tar -czf sql-lint-service-offline.tar.gz \
  sql-lint-service/ \
  offline-packages/

# 检查包大小
ls -lh sql-lint-service-offline.tar.gz
```

## 三、离线环境部署步骤

### 3.1 传输文件到目标服务器
```bash
# 方法1：SCP传输
scp sql-lint-service-offline.tar.gz user@target-server:/home/user/

# 方法2：U盘拷贝
# 将tar.gz文件拷贝到U盘，再复制到目标服务器用户目录
```

### 3.2 目标服务器环境准备
```bash
# 登录目标服务器
ssh user@target-server

# 进入用户目录
cd ~

# 解压部署包
tar -xzf sql-lint-service-offline.tar.gz
cd sql-lint-service

# 检查Python版本
python3 --version
# 应为 Python 3.12.x

# 如无Python3.12，需先安装（离线安装方法见附录A）
```

### 3.3 安装Python虚拟环境
```bash
# 安装virtualenv（如未安装）
# 离线安装方法见附录B

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 验证虚拟环境
which python
python --version
```

### 3.4 安装离线依赖
```bash
# 使用本地包安装依赖
pip install --no-index \
  --find-links=./offline-packages \
  -r requirements.txt

# 验证安装
pip list | grep -E "(sqlfluff|fastapi|uvicorn)"
# 应显示：
# fastapi     0.135.3
# sqlfluff    4.1.0
# uvicorn     0.44.0
```

### 3.5 配置服务

#### 3.5.1 创建用户级systemd服务
```bash
# 创建用户systemd目录
mkdir -p ~/.config/systemd/user/

# 创建用户级systemd服务文件
cat > ~/.config/systemd/user/sql-lint.service << 'EOF'
[Unit]
Description=SQL Lint Service (User)
After=network.target

[Service]
Type=simple
WorkingDirectory=/home/%u/sql-lint-service
Environment="PATH=/home/%u/sql-lint-service/venv/bin"
ExecStart=/home/%u/sql-lint-service/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=default.target
EOF
```

#### 3.5.2 创建启动脚本（备选方案）
```bash
# 创建启动脚本
cat > ~/start-sql-lint.sh << 'EOF'
#!/bin/bash
cd /home/$USER/sql-lint-service
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
EOF

chmod +x ~/start-sql-lint.sh
```

#### 3.5.3 创建环境配置文件
```bash
# 创建.env文件
cat > ~/sql-lint-service/.env << 'EOF'
# SQL Lint Service 配置
ENABLE_HOT_RELOAD=false
LOG_LEVEL=INFO
HOST=0.0.0.0
PORT=8000
EOF

chmod 600 ~/sql-lint-service/.env
```

### 3.6 启动服务

#### 方案A：使用用户级systemd服务
```bash
# 启用用户级systemd服务
systemctl --user daemon-reload
systemctl --user enable sql-lint.service
systemctl --user start sql-lint.service

# 检查服务状态
systemctl --user status sql-lint.service

# 设置用户服务开机自启（需要启用linger）
sudo loginctl enable-linger $USER
```

#### 方案B：使用screen/tmux后台运行
```bash
# 安装screen（如未安装）
# sudo yum install screen  # CentOS
# sudo apt install screen  # Ubuntu

# 创建screen会话
screen -S sql-lint

# 在screen会话中启动服务
cd ~/sql-lint-service
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 退出screen会话（服务继续运行）
# 按 Ctrl+A, 然后按 D

# 重新连接screen会话
screen -r sql-lint
```

#### 方案C：使用nohup后台运行
```bash
# 创建启动脚本
cat > ~/sql-lint-service/start.sh << 'EOF'
#!/bin/bash
cd /home/$USER/sql-lint-service
source venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &
echo $! > app.pid
echo "Service started with PID: $(cat app.pid)"
EOF

chmod +x ~/sql-lint-service/start.sh

# 创建停止脚本
cat > ~/sql-lint-service/stop.sh << 'EOF'
#!/bin/bash
cd /home/$USER/sql-lint-service
if [ -f app.pid ]; then
    kill $(cat app.pid)
    rm app.pid
    echo "Service stopped"
else
    echo "No running service found"
fi
EOF

chmod +x ~/sql-lint-service/stop.sh

# 启动服务
~/sql-lint-service/start.sh

# 查看日志
tail -f ~/sql-lint-service/app.log
```

```bash

/home/$USER/sql-lint-service/app.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 644 $USER $USER
}

```

```bash
#!/bin/bash
# deploy.sh - 一键部署脚本

USER_HOME=$HOME

echo "开始部署SQL Lint Service到用户目录..."

# 1. 解压部署包
if [ -f "sql-lint-service-offline.tar.gz" ]; then
    echo "解压部署包..."

    echo "错误: 未找到部署包 sql-lint-service-offline.tar.gz"

# 2. 创建虚拟环境

cd $SERVICE_DIR
python3 -m venv venv

# 3. 安装依赖
echo "安装离线依赖..."
source venv/bin/activate
pip install --no-index --find-links=./offline-packages -r requirements.txt

# 4. 创建配置文件
echo "创建配置文件..."
cat > .env << 'EOF'
ENABLE_HOT_RELOAD=false
LOG_LEVEL=INFO
HOST=0.0.0.0

# 5. 创建启动脚本
echo "创建启动脚本..."
cat > start.sh << 'EOF'
#!/bin/bash
cd $SERVICE_DIR
source venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &
echo $! > app.pid
echo "SQL Lint Service started with PID: $(cat app.pid)"
EOF

chmod +x start.sh

cat > stop.sh << 'EOF'
#!/bin/bash
cd $SERVICE_DIR
if [ -f app.pid ]; then
    kill $(cat app.pid)
    rm app.pid
    echo "Service stopped"
else
    echo "No running service found"

chmod +x stop.sh

echo "部署完成！"
echo "启动服务: $SERVICE_DIR/start.sh"
echo "停止服务: $SERVICE_DIR/stop.sh"
echo "查看日志: tail -f $SERVICE_DIR/app.log"
```

### 附录D：健康检查脚本（用户目录版）
```bash
#!/bin/bash
# health-check.sh

SERVICE_URL="http://localhost:8000"
LOG_FILE="$HOME/sql-lint-service/app.log"

check_service() {
    # 检查进程
    if [ -f "$HOME/sql-lint-service/app.pid" ]; then
        pid=$(cat "$HOME/sql-lint-service/app.pid")
        if ps -p $pid > /dev/null 2>&1; then
            echo "✓ 服务进程运行中 (PID: $pid)"
            return 0
        else
            echo "✗ 服务进程不存在"
            return 1
        fi
    else
        echo "✗ 未找到PID文件"
        return 1
    fi
}

check_health() {
    response=$(curl -s -o /dev/null -w "%{http_code}" "${SERVICE_URL}/health" 2>/dev/null)
    if [ "$response" = "200" ]; then
        echo "✓ 服务健康检查通过"
        return 0
    else
        echo "✗ 服务健康检查失败 (HTTP $response)"
        return 1
    fi
}

check_logs() {
    if [ -f "$LOG_FILE" ]; then
        errors=$(tail -20 "$LOG_FILE" | grep -i "error\|exception\|traceback")
        if [ -z "$errors" ]; then
            echo "✓ 日志无错误"
            return 0
        else
            echo "✗ 发现错误日志:"
            echo "$errors"
            return 1
        fi
    else
        echo "⚠ 未找到日志文件"
        return 2
    fi
}

# 执行检查
echo "=== SQL Lint Service 健康检查 ==="
check_service
check_health
check_logs
```

## 部署总结
1. **路径**: 所有文件部署在 `/home/$USER/sql-lint-service/`
2. **权限**: 使用当前用户权限，无需root
3. **服务管理**: 提供多种方案（用户级systemd、screen、nohup）
4. **维护**: 包含完整的启动、停止、监控脚本
5. **离线**: 完全离线部署，无需互联网连接

此方案适合在无root权限的用户目录下部署SQL Lint Service。
